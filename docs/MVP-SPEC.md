## 一、MVP 必须包含的 5 个核心功能

### F1：多项目管理（Multi-Project Management）

**功能描述**

用户在 DeerFlow-X 上可同时管理多个独立项目，每个项目完全隔离，拥有独立的配置、Agent、环境和部署目标。项目之间不共享状态，支持快速切换。

**详细规格**

- 创建项目：输入项目名称、描述、技术栈（Node/Python/Go等）
- 项目列表：展示所有项目卡片（名称、最近活动时间、Agent数量）
- 项目隔离：每个项目的配置文件、密钥、Agent定义完全独立
- 项目删除：软删除（30天内可恢复）

**边界条件**

- Free用户最多3个项目
- 项目名全局唯一，不可重复

---

### F2：Agent 团队调度（Agent Team Orchestration）

**功能描述**

DeerFlow-X 支持在同一项目内定义多个 Agent，组成协作团队。用户可指定 Agent 角色（规划者/执行者/审查者/观测者），定义任务执行顺序（串行/并行/条件分支）。

**详细规格**

- Agent 定义：名称、角色、模型选择（GPT-4/Claude-3/Gemini）、系统提示词
- 角色预设：
  - **Planner**：负责任务拆解和规划
  - **Executor**：执行具体操作（写代码、调用API、搜索）
  - **Reviewer**：审查 Executor 输出，提出修改意见
  - **Observer**：监控执行过程，记录日志
- 调度策略：串行执行、并行并行、条件分支（if-else/loop）
- 任务队列：支持暂停、恢复、取消、重试

**边界条件**

- 单项目最多同时运行10个任务
- 单个Agent执行超时默认10分钟，可配置
- Agent间消息传递通过共享上下文

---

### F3：本地开发环境（Local Development Environment）

**功能描述**

DeerFlow-X 提供本地CLI工具，将Agent能力带入用户本地开发环境。用户在本地编辑器中工作，Agent在后台协作，结果实时同步。

**详细规格**

- CLI安装：`npx deerflow-x init` 一键初始化
- 本地连接：通过WebSocket与云端服务保持连接
- 文件同步：Agent修改文件后自动diff，用户确认后生效
- 交互模式：
  - **Chat模式**：用户自然语言描述需求，Agent团队协作完成
  - **Agent模式**：Agent自动完成任务，用户仅做最终审批
- 终端集成：在终端中实时显示Agent执行日志

**边界条件**

- 不支持纯离线模式（需保持连接）
- 大文件（>5MB）跳过同步
- Git未初始化目录需先init

---

### F4：一键部署（One-Click Deployment）

**功能描述**

DeerFlow-X 将项目代码自动构建并部署到目标平台，支持主流Serverless/容器平台，零配置一键完成。

**详细规格**

- 支持平台：Vercel（优先）、Railway、Render、Self-hosted Docker
- 部署流程：
  1. 自动检测技术栈（Node/Python/Go等）
  2. 生成/使用现有 Dockerfile 或构建脚本
  3. 构建镜像并推送到目标平台
  4. 自动配置环境变量和域名
- 预览环境：每个PR自动生成预览链接，PR关闭后自动销毁
- 回滚：支持一键回滚到上一个稳定版本
- 部署日志：实时流式日志输出

**边界条件**

- 构建超时：默认20分钟
- 私有仓库需提前配置SSH Key
- 单项目最多3个并发部署任务

---

### F5：GitHub OAuth 登录与集成

**功能描述**

用户通过 GitHub 账号登录，DeerFlow-X 自动同步用户仓库列表，支持直接从仓库导入项目。

**详细规格**

- OAuth登录：GitHub账号一键登录（支持邮箱/用户名密码备用）
- 仓库同步：读取用户/组织仓库列表（需用户授权）
- 项目导入：从指定仓库导入，自动生成Agent配置模板
- Webhook集成：监听Push/PR事件，触发Agent自动执行CI任务
- 部署凭证：安全存储GitHub Token，不明文暴露

**边界条件**

- 需要GitHub账号（不支持其他OAuth平台）
- 组织仓库需要组织管理员授权
- Token有效期30天，需用户手动刷新

## 三、用户故事（3个典型场景）

### 用户故事 1：副业创业者小王，4小时上线MVP

**背景**：小王白天是某互联网公司的后端工程师，晚上做副业——他想做一个AI简历优化工具。预计市场规模不大，但想快速验证。

**典型流程**：
1. 小王用GitHub账号登录DeerFlow-X，创建项目"resume-optimizer"
2. 定义3个Agent团队：Planner（分析需求）、Executor（写前后端代码）、Reviewer（Code Review）
3. 在Chat里输入："帮我做一个简历优化工具，用户上传简历PDF，AI分析并给出优化建议"
4. Planner拆解任务 → Executor生成Next.js前端 + FastAPI后端 → Reviewer审查代码
5. 点击"一键部署到Vercel"，3分钟后获得可访问URL
6. 把链接发到Twitter，看用户反馈

**结果**：从想法到线上demo，小王只用了4小时。如果不用DeerFlow-X，他需要自己搭环境、写代码、配CI，至少2天。

**关键指标**：
- 上线时间：4小时（vs业界平均48小时）
- 部署成功率：>95%
- 用户满意度：NPS > 50

### 用户故事 2：独立开发者老张，一个人的AI SaaS

**背景**：老张45岁，资深全栈工程师，厌倦了打工，决定自己做一个数据分析SaaS，靠订阅费生活。他需要多租户计费、监控、运维自动化——这些通常需要团队。

**典型流程**：
1. 老张升级到Team套餐（$49/月），创建项目"data-insights"
2. 定义Agent团队：DataParser（数据清洗）、Analyzer（统计分析）、Reporter（生成报告）、BillingMonitor（监控订阅状态）
3. 用DeerFlow-X内置的Stripe集成，5分钟配置好订阅计费（$9.9/月 Starter / $29.9/月 Pro）
4. 上线第一周，获得20个付费用户
5. 用户A反馈数据导出慢，老张让Analyzer Agent自动优化查询，同时不影响其他用户

**结果**：老张一个人干了以前需要3-4人团队的事情。他的SaaS月收入$500，覆盖DeerFlow-X成本绰绰有余。

**关键指标**：
- 月收入：$500+
- 用户续费率：>80%
- 运维时间：<1小时/周

### 用户故事 3：独立开发者小林，CLI重度用户的效率革命

**背景**：小林是终端重度用户，不喜欢图形界面。他在VSCode终端里完成所有工作，想要Agent能力但不想切换工具。

**典型流程**：
1. 小林在项目目录运行 `npx deerflow-x init`
2. 配置Agent团队，写入 `.deerflow/agents.yaml`
3. 在终端输入 `deerflow run "重构这个API，改成RESTful风格，添加分页"`
4. Executor Agent读取代码 → 分析 → 修改 → Reviewer审查 → 自动生成测试
5. 查看diff，敲 `y` 确认所有更改
6. `deerflow deploy` 一键发布到Railway

**结果**：小林全程没有离开终端，工作效率提升3倍。

**关键指标**：
- CLI日活用户占比：>60%
- 平均任务完成时间：<5分钟
- 用户留存率（30天）：>70%
